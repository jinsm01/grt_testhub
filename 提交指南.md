# Git 提交指南

## 手动提交步骤

### 1. 打开 PowerShell
按 `Win + X`，选择 `Windows PowerShell` 或 `终端`

### 2. 进入项目目录
```powershell
cd D:\testhub\grt_testhub
```

### 3. 检查 Git 状态
```powershell
git status
```

### 4. 初始化 Git 仓库（如果还没有初始化）
```powershell
git init
```

### 5. 添加远程仓库
```powershell
git remote remove origin
git remote add origin https://github.com/jinsm01/grt_testhub.git
```

### 6. 添加所有修改的文件
```powershell
git add .
```

### 7. 提交更改
```powershell
git commit -m "优化前端页面样式

- 优化 ReviewDetail.vue 样式，参考 ReviewList.vue 设计风格
- 优化 ReviewForm.vue 页面样式，统一紫色主题
- 修复 TestCaseEdit.vue 表单项间距问题
- 修复 ReviewForm.vue 对话框按钮样式
- 修复 ReviewTemplateList.vue 对话框按钮样式和表单间距
- 统一对话框样式，使用 dialog-footer class"
```

### 8. 推送到远程仓库
```powershell
git push -u origin master
```

如果提示输入用户名和密码，请输入你的 GitHub 用户名和个人访问令牌（Token）。

## 常见问题

### 1. 如果提示 "fatal: not a git repository"
说明还没有初始化 Git 仓库，请执行：
```powershell
git init
```

### 2. 如果提示 "fatal: remote origin already exists"
说明远程仓库已经存在，请执行：
```powershell
git remote remove origin
git remote add origin https://github.com/jinsm01/grt_testhub.git
```

### 3. 如果推送失败，提示权限错误
请检查：
- 是否已登录 GitHub
- 是否使用个人访问令牌（Token）而不是密码
- 是否有仓库的写入权限

### 4. 如果需要查看修改了哪些文件
```powershell
git diff --name-only
```

### 5. 如果需要查看具体的修改内容
```powershell
git diff
```
