## 手动推送步骤

### 方法 1：使用 PowerShell 推送（推荐）

1. 打开 PowerShell
2. 进入项目目录：
   ```powershell
   cd D:\testhub\grt_testhub
   ```

3. 配置 Git 用户信息（如果还没配置）：
   ```powershell
   git config user.email "your-email@example.com"
   git config user.name "Your Name"
   ```

4. 添加远程仓库：
   ```powershell
   git remote add origin https://github.com/jinsm01/grt_testhub.git
   ```

5. 推送到 GitHub：
   ```powershell
   git push -u origin master
   ```

6. 输入 GitHub 用户名和密码/Token

### 方法 2：使用 GitHub Token 推送

1. 生成 GitHub 个人访问令牌：
   - 访问 https://github.com/settings/tokens
   - 点击 "Generate new token"
   - 选择 `repo` 权限
   - 生成并复制令牌

2. 使用 Token 推送：
   ```powershell
   cd D:\testhub\grt_testhub
   git remote set-url origin https://YOUR_TOKEN@github.com/jinsm01/grt_testhub.git
   git push -u origin master
   ```

### 方法 3：使用 GitHub Desktop

1. 下载并安装 GitHub Desktop：https://desktop.github.com/
2. 打开 GitHub Desktop
3. 选择 "Add existing repository"
4. 选择 `D:\testhub\grt_testhub` 目录
5. 点击 "Publish repository"
6. 登录 GitHub 账号并推送

### 方法 4：使用 VS Code

1. 打开 VS Code
2. 打开项目文件夹 `D:\testhub\grt_testhub`
3. 点击左侧的源代码管理图标
4. 点击 "发布到 GitHub"
5. 按照提示完成推送

## 常见问题

### 1. 提示 "fatal: not a git repository"
说明当前目录不是 Git 仓库，请先执行：
```powershell
git init
```

### 2. 提示 "remote origin already exists"
说明远程仓库已存在，请先移除再添加：
```powershell
git remote remove origin
git remote add origin https://github.com/jinsm01/grt_testhub.git
```

### 3. 提示 "Permission denied"
说明没有权限推送，请检查：
- 是否有仓库的写入权限
- 用户名和密码/Token 是否正确

### 4. 提示 "Could not resolve host"
说明网络连接问题，请检查：
- 网络连接是否正常
- 是否配置了代理
- 防火墙是否阻止了连接
